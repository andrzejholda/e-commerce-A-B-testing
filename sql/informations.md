## SQL Architecture Overview

This project follows a layered SQL transformation approach designed to ensure:
- clear data lineage
- controlled analytical granularity
- modular transformation logic
- reproducible product analytics metrics
- scalable experimentation analysis

The SQL layer transforms raw product interaction data into structured analytical datasets used in the Power BI product analytics dashboard.

The SQL architecture is divided into three complementary analytical models:
- 1️⃣ Product Funnel & Conversion Model
- 2️⃣ Cohort Retention & Lifecycle Model
- 3️⃣ Experimentation (A/B Testing) Model

Each model supports a different aspect of product analytics and business decision-making.

## 1️⃣ Product Funnel & Conversion Model
### Objective
Analyze user behavior across the product funnel and identify where users drop off before completing a purchase.

This model supports:
- funnel performance monitoring
- conversion rate optimization
- UX improvement initiatives
- drop-off analysis across funnel stages

The funnel stages analyzed in this project are:

app_open → view_product → add_to_cart → purchase

#### 01_funnel_events.sql — Event Aggregation Layer
##### Purpose

Transform raw event-level tracking data into a structured dataset suitable for funnel analysis.
This layer aggregates product interaction events at the daily level to measure how many users reach each funnel stage.

Output
- funnel_events

Grain
One row per:
event_type per day

Key Logic
- Counts distinct users reaching each event stage
- Groups events by date for time-based analysis
- Prepares dataset for funnel visualizations in Power BI

This dataset powers:
- funnel charts
- conversion trend analysis
- drop-off visualization

#### 02_user_funnel.sql — User-Level Funnel Model
##### Purpose

Transform event-level tracking data into a user-level behavioral model representing whether a user reached each stage of the funnel.
This enables accurate conversion rate calculations between funnel steps.

Output
- user_funnel

Grain
One row per user

Key Logic
- Uses conditional aggregation to detect whether a user performed each funnel action.

Each user receives binary indicators for:
- app_open
- view_product
- add_to_cart
- purchase

This structure enables:
- funnel conversion analysis
- behavioral segmentation
- experiment impact analysis

## 2️⃣ Monetization Metrics Model
### Objective

Quantify revenue generation and connect user behavior with monetization outcomes.

This model supports:
- ARPU analysis
- revenue segmentation by country and device
- monetization comparison across experiment variants

#### 01_user_revenue.sql — Revenue Aggregation Layer
#####Purpose

Aggregate purchase transactions at the user level to calculate revenue metrics.

Output
- revenue_user

Grain
One row per user

Key Logic
Aggregates:
- total revenue generated by each user
- number of purchases per user

This dataset enables calculation of:
- ARPU (Average Revenue Per User)
- ARPPU (Average Revenue Per Paying User)

#### 02_user_metrics.sql — Enriched User Metrics Layer
##### Purpose

Combine user attributes with monetization metrics to support segmentation analysis.

Output
- user_metrics

Grain
One row per user

Key Logic
Joins user-level revenue data with user attributes such as:
- country
- device
- experiment variant

This dataset powers:
- revenue segmentation dashboards
- geographic performance comparisons
- device-level performance analysis

## 3️⃣ Cohort & Lifecycle Retention Model
### Objective

Analyze long-term user engagement by tracking how user activity evolves after registration.

This model enables:
- cohort retention analysis
- lifecycle engagement tracking
- early churn detection
- product stickiness evaluation

#### 01_cohort_base.sql — Cohort Assignment
##### Purpose

Assign each user to a cohort based on their registration month.

Output
- user_cohort

Grain
One row per user

Key Logic
Defines:
cohort_month = DATE_TRUNC('month', registration_date)

This establishes the starting point of the user lifecycle.

#### 02_user_activity.sql — Monthly Activity Layer
##### Purpose

Track monthly user engagement by identifying the months in which users generated events.

Output
- user_activity

Grain
One row per user per activity month

Key Logic
Aggregates event-level data into monthly activity records.

This dataset represents user engagement over time.

#### 03_cohort_activity.sql — Lifecycle Calculation
##### Purpose

Calculate the lifecycle stage of each user relative to their cohort start.

Output
- cohort_activity

Grain
One row per user per cohort lifecycle month

Key Logic
Calculates:
cohort_age = months between activity_month and cohort_month

Example:

M0 → registration month
M1 → first month after registration
M2 → second month after registration

This enables lifecycle-based retention analysis.

#### 04_cohort_retention.sql — Retention Aggregation
##### Purpose

Aggregate active users by cohort and lifecycle stage.

Output
- cohort_retention

Grain
One row per cohort month per lifecycle month

Key Logic
Counts distinct active users for each:
- cohort_month
- cohort_age


This dataset powers the cohort retention matrix in Power BI.
It enables analysis of:
- retention curves
- long-term engagement
- product stickiness

## 4️⃣ Experimentation (A/B Testing) Model
### Objective

Evaluate the impact of product changes using controlled A/B experimentation.

This model supports:
- experiment performance evaluation
- conversion rate comparison
- revenue impact estimation

#### 01_ab_test_summary.sql — Experiment Performance
##### Purpose

Aggregate key business metrics by experiment variant.

Output
- ab_test_summary

Grain
One row per experiment variant

Key Logic
Calculates:
- number of users in each variant
- number of purchases
- total revenue

This dataset supports:
- conversion comparison
- revenue uplift analysis

#### 02_ab_conversion.sql — Conversion Analysis
##### Purpose

Measure conversion performance between experiment variants.

Output
- ab_conversion

Grain
One row per experiment variant

Key Logic
Counts users who:
- viewed a product
- completed a purchase


This dataset enables calculation of:
- conversion rate
- conversion uplift
- estimated revenue impact
